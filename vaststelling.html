Mijn vaststellingen (wat misschien zou kunnen zijn...)

Onderwerp: Analyse dataverlies bij inschrijfformulieren

1. Context & Vaststelling

Bij de infosessie "Bewegen en diabetes" is gebleken dat enkele deelnemers niet op de definitieve Excel-lijst stonden, terwijl zij wel een automatische bevestigingsmail hadden ontvangen. Dit bewijst dat de browser van de gebruiker de verzendopdracht heeft uitgevoerd, maar dat de server de data niet heeft opgeslagen.

2. Analyse Browser Console (F12)

Vermits het inschrijvingsformulier "Bewegen en diabetes" niet meer online staat, heb ik een analyse gedaan van het huidige formulier (Wandeling Bolderberg) wat vermoedelijk dezelfde fouten bevat.

Gevonden Error 1: Mixed Content Blokkade
Mixed Content: The page at 'https://www.dlml.be/wandeling/index.html' was loaded over HTTPS, but requested an insecure script 'http://code.jquery.com/jquery-latest.js'. This request has been blocked; the content must be served over HTTPS.

Impact: Moderne iPhones (Safari) toestellen blokkeren dit script onmiddellijk. Omdat dit script de motor is voor de data-overdracht naar de lijst, wordt de inschrijving simpelweg gestopt.

Waarom verschijnt er geen foutmelding?
Browsers blokkeren deze onveilige scripts stilletjes op de achtergrond. De JavaScript-code die normaal gesproken een foutmelding op het scherm zou tonen, is afhankelijk van dit geblokkeerde script en kan dus zelf ook niet meer worden uitgevoerd. Voor de gebruiker (en het systeem) lijkt het proces dus een 'stille dood' te sterven zonder dat er een alarmbel afgaat.

Gevonden Error 2: Resource Missing
jquery.selection.js:1 Failed to load resource: the server responded with a status of 404 ()

Impact: Het ontbreken van dit bestand veroorzaakt JavaScript-crashes tijdens het klikproces, wat onvoorspelbaar gedrag van de verzendknop tot gevolg heeft.

Hoe kunt ge dit zelf controleren?

Moest ge de Console-functie niet kennen, dan kunt ge deze foutmeldingen eenvoudig zelf bekijken op uw computer:

  • Open het formulier in uw browser (bijv. Google Chrome of Edge).
  • Druk op uw toetsenbord op de toets F12.
  • Klik bovenaan in het venster dat rechts (of onderaan) verschijnt op het tabblad 'Console'.
  • U ziet hier de rode tekst onder het onderdeel 'Errors'. Dit is wat de browser van een deelnemer ziet wanneer de pagina geladen wordt.

3. De Oorzaak: Verouderde Techniek

Technisch feit:

De website van de DLML werkt keurig en veilig via HTTPS. Echter wordt er in de code een essentieel script aangeroepen via het verouderde HTTP (zonder de 's').

Dit creëert een situatie van "onveilig binnen veilig". Moderne smartphones accepteren dit niet meer en blokkeren de verbinding met het script om de gebruiker te beschermen.

De gebruikte link http://code.jquery.com/jquery-latest.js is bovendien door de ontwikkelaars al in 2014 bevroren op versie 1.11.1. Deze software is 12 jaar oud en niet langer compatibel met de strenge beveiligingseisen van smartphones in 2026.

4. Het voorstel (wat misschien kan helpen)

De 5 verouderde script-regels in de HTML-formulier(en) vervangen door 2 iets modernere, veilige HTTPS-regels.

Stap A: Verwijder deze 5 regels volledig:

<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script> <script src="https://code.jquery.com/jquery.min.js"> </script> <script src="http://code.jquery.com/jquery-latest.js"></script> <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery.inputmask/3.3.4/jquery.inputmask.bundle.min.js"></script> <script src="jquery.selection.js"></script>

Stap B: Plaats deze 2 iets modernere regels in de plaats:

<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.7.1/jquery.min.js"></script> <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery.inputmask/3.3.4/jquery.inputmask.bundle.min.js"></script>

5. Verificatie of het geholpen heeft

  1. Voer de bovenstaande wijziging door in de HTML.
  2. Open de HTML-pagina in een browser en gebruik de F12-check zoals hierboven beschreven.
  3. Indien nodig herlaad de pagina (F5). De rode errors kunnen nu misschien volledig verdwenen zijn.
