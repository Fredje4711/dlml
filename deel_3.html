1. Inschrijvingsformulier (leefstijl/index.html)

Bestand: leefstijl/index.html
Huidige code (met kleine typfout):
<met77a http-equiv="X-UA-Compatible" content="IE=Edge">
Aangepaste code:
<meta http-equiv="X-UA-Compatible" content="IE=Edge">
Vaststelling: Waarschijnlijk is er een klein typfoutje geslopen bij het aanpassen. De browser herkent 'met77a' niet. Door dit te herstellen, werkt de instructie voor de weergave-motor op Windows-computers weer correct.
Huidige script-regels (3 regels):
<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.7.1/jquery.min.js"></script> <script src="https://code.jquery.com/jquery.min.js"> </script> <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery.inputmask/3.3.4/jquery.inputmask.bundle.min.js"></script>
Aangepaste script-regels (2 regels):
<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.7.1/jquery.min.js"></script> <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery.inputmask/3.3.4/jquery.inputmask.bundle.min.js"></script>
Vaststelling: De tweede regel (code.jquery.com) is inmiddels HTTPS, dus die geeft geen blokkade meer. Maar omdat de regel erboven (ajax.googleapis.com) exact dezelfde motor inlaadt, is die tweede regel overbodig geworden.
Waarom is verwijderen beter?
Twee keer dezelfde bibliotheek laden is als twee kapiteins op één schip: het maakt de pagina onnodig zwaar voor de gsm van de bezoeker en het kan soms voor kleine conflicten zorgen tussen de scripts. Door er eentje te houden, blijft de code schoon, snel en stabiel.

2. Web-app configuratie (Aanpassing Manifest & Icoontjes)

Om te zorgen dat men een mooi DLML-icoontje op de gsm kan krijgen, moeten we het manifest-bestand op twee plaatsen in orde zetten. Let op: het is belangrijk dat zowel de bestandsnaam als de verwijzing in de code gewijzigd wordt.

Locatie 1: De Hoofdmap (./)

A. Actie in de hoofdmap:

Hernoem het bestand site.webmanifest.html naar site.webmanifest

B. Actie in de code van ./index.html:

Huidige code:
<link rel="manifest" href="site.webmanifest.html">
Nieuwe code:
<link rel="manifest" href="site.webmanifest">

Locatie 2: De map van het formulier (leefstijl/)

A. Actie in de map "Leefstijl":

Hernoem het bestand in deze map ook van site.webmanifest.html naar site.webmanifest

B. Actie in de code van leefstijl/index.html:

Huidige code:
<link rel="manifest" href="site.webmanifest.html">
Nieuwe code:
<link rel="manifest" href="site.webmanifest">
Vaststelling: Een manifest-bestand is een configuratiebestand voor smartphones en geen webpagina. Door de extensie .html overal te verwijderen, herkent een iPhone dit direct als de officiële 'App-instellingen' van onze site.

4. Iconen Mythen & Feiten (site/css/diabetesinfo.css)

Bestand: site/css/diabetesinfo.css

Omdat we nu FontAwesome 6 gebruiken, verandert de browser de icoontjes automatisch in SVG-plaatjes. De huidige CSS kijkt alleen naar het "i-element", waardoor de nieuwe icoontjes klein in de hoek blijven staan. We moeten de svg toevoegen aan de stijlregels op drie plaatsen:

AANPASSING 1: Standaard weergave (GSM)
.card_front i { width:100%; height:100%; grid-column:2; grid-row: 5; text-align:center; font-size: 10vw; line-height:1; }
.card_front i, .card_front svg { width:100%; height:100%; grid-column:2; grid-row: 5; text-align:center; font-size: 10vw; line-height:1; }
AANPASSING 2: @media only screen and (min-width: 768px)
.card_front i { font-size: 7vw; line-height:1; }
.card_front i, .card_front svg { font-size: 7vw; line-height:1; }
AANPASSING 3: @media only screen and (min-width: 1280px)
.card_front i { font-size: 7vw; line-height:1; }
.card_front i, .card_front svg { font-size: 7vw; line-height:1; }
Vaststelling: Door overal , .card_front svg toe te voegen aan de selector, begrijpt de browser dat de nieuwe SVG-iconen exact dezelfde grootte (10vw of 7vw) en positie (grid-row 5) moeten krijgen als de oude lettertype-icoontjes. Hierdoor staan ze weer netjes groot en gecentreerd op de kaarten.
